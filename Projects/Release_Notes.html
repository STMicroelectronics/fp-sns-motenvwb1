Purpose

The FP-SNS-MOTENVWB1 Projects provide applications to aid understanding how to develop an application using the STM32_WPAN Middleware and Motion libraries middlewares. Projects include STM32CubeMX project file (.ioc) for the graphical visualization of the STM32 microcontroller pins, peripherals and supported expansion nucleo boards. STM32CubeMX project files (.ioc) can be used for generate the MOTENV1 and MOTENV1_OTA application for each supported board.

Development Toolchains and Compilers

  • IAR Embedded Workbench for ARM (EWARM) toolchain V9.20.1
  • RealView Microcontroller Development Kit (Open Keil µVision 5 - MDK-ARM) toolchain V5.37.0
  • STM32CubeIDE V1.12.1

Supported Devices and Boards

  • P-NUCLEO-WB55RG Nucleo-68 board with STM32WB55 MCU supporting Bluetooth™ 5 and 802.15.4 connectivity
  • X-NUCLEO-IKS01A3 Expansion board for four MEMS sensor devices:
    • HTS221, LPS22HH, LSM6DSO, LIS2MDL
  • X-NUCLEO-IKS4A1 Expansion board for four MEMS sensor devices:
    • STTS22H, SHT40AD1B, LPS22DF, LSM6DSV16X, LIS2MDL
  • X-NUCLEO-53L3A2 Multi-targets ranging sensor expansion board based on VL53L3CX for STM32 Nucleo (Optional).

Issues

  • After code generation MOTENV1 from STM32CubeMX tool for STM32CubeIDE toolchain, after building if the generated binary doesn’t work, replace the linker scripts with the scripts in the folder LinkerScript.

  • After code generation MOTENV1_OTA from STM32CubeMX software before building:

    • for IAR, Keil and STM32CubeIDE replace the linker scripts with the scripts in the folder LinkerScript
    • for Keil, open the “Options for Target” and in the tab “Target” set start= 0x8007000 and sixe= 0x79000.

Very Iimportant

MOTENV1_OTA allow the Firmware-Over-The-Air (FOTA). To support OTA you need:

  1. to flash BLE_Ota_reference.hex from 0x8000000 at the beginning of the FLASH (address 0x08000000)
  2. flash user application from 0x8007000
